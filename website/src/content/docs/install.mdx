## Preferred install path

If you are on a supported platform, the fastest path is the published npm package:

```bash
npm install -g @olhapi/maestro
```

Update an existing global install with:

```bash
npm update -g @olhapi/maestro
```

The installed command name is still `maestro`.

Official npm builds currently cover:

- macOS arm64
- macOS x64
- Linux x64 (glibc)
- Linux arm64 (glibc)
- Windows x64

Linux npm packages target glibc only. Alpine and other musl-based distros should build from source or use Docker.

## Build from source

For local development or unsupported platforms, build the binary directly from the repo:

```bash
go build -o maestro ./cmd/maestro
```

Optional Docker build:

```bash
docker build -t maestro .
```

## Contributor tooling

If you are working in the repo itself, install the root development dependency once:

```bash
npm install
```

That installs the repo-managed Git hooks and bootstraps the nested frontend and website dependencies used by those hooks.

## What to install first

- Install `go` if you need source builds or local development.
- Install `codex` before you attempt real orchestration runs that launch the Codex CLI.
- Use the npm package when you want the shortest path to a working local install on a supported platform.
